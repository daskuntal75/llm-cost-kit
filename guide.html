The Foundation

8 principles that apply on every platform.

The same waste patterns appear in Claude, ChatGPT, and Gemini. The platform-specific files in this kit are just different implementations of the same ideas.

1
Information Density Over Brevity
The goal is not short responses — it's responses where every token carries signal. A complete 400-token answer beats a 200-token answer that needs a follow-up, because follow-ups re-send the whole thread.
2
Context Is Expensive At Rest
Prompt cache hits are cheap. Cache misses are full price. Summarize before switching topics. The smaller the context you carry into a new session, the cheaper every subsequent turn.
3
Match Model to Task
Most tasks don't require the best model. Route cheap work to small models. Escalate to top-tier only after the default model fails twice on the same task, same session.
4
Tools Are Not Free
Every tool definition loaded — MCP server, ChatGPT plugin, Gemini extension, API function — adds its full schema to every input message. Load only what the current task will actually call.
5
Sub-Agents Need Minimum Viable Context
Full conversation passed to N parallel agents = N× the thread cost. Orchestrator holds full state. Sub-agents get a scoped JSON brief with 5 fields.
6
Thread Hygiene Is Maintenance, Not Cleanup
Letting a thread run to 30 turns is technical debt. Summarize and reset at turn 15 — it costs almost nothing and saves everything important.
7
Output Discipline Saves Tokens on Every Response, Forever
Setting user preferences once eliminates 20–60 tokens of pure overhead from every response, every conversation, permanently. Highest ROI move in the entire kit.
8
Measure, Then Optimize
Token optimization without visibility is guessing. Review your usage dashboard on the 1st of each month. Which model drove most cost? Was it justified?

Claude — Key Differentiator

The /compact vs /clear timing rule (5-minute cache window) is Claude-specific and the highest-precision cost lever in the kit. On other platforms, manual summarize + new chat is the only mechanism.

OpenAI — Key Differentiator

o1/o3 reasoning models charge for internal reasoning tokens invisible in the output. A single hard o3 task can cost more than 50 gpt-4o calls. The escalation threshold here is stricter: 2 failures AND the task is genuinely reasoning-heavy.

Gemini — Key Differentiator

1M+ token context windows are real — but long context is not free context on the API. Gems are the native scoping unit (one Gem per context area) which makes project isolation more natural than on the other platforms.

Setup Guide

One config file. One wizard. Three platforms.

1
automated

Edit config.yaml — the only file you need to configure

Open config.yaml and uncomment the platforms you use. Fill in your project contexts. Set your preferences. This is the single source of truth — all platform adapters read from it.

config.yaml — the one file to edit
# Uncomment the platforms you use:
platforms:
  enabled:
    - claude        # Claude Chat · Claude Code · Cowork
    - openai        # ChatGPT · Custom GPTs · OpenAI API
    # - gemini      # Gemini · Gems · Google AI Studio

# Universal preferences (applied to all platforms):
response_style:
  lead_with_answer: true
  no_openers: true
  no_closers: true
  tables_over_prose: true

# Model routing (one entry per platform):
model_routing:
  claude:
    default: claude-sonnet-4-6
    sub_tasks: claude-haiku-4-5-20251001
    escalate: claude-opus-4-6
  openai:
    default: gpt-4o
    sub_tasks: gpt-4o-mini
    escalate: o3
2
automated

Run the unified setup wizard

The wizard reads your config.yaml and runs only the adapters for enabled platforms. Idempotent — safe to re-run after adding a new platform.

shell
git clone https://github.com/[your-username]/llm-cost-kit.git
cd llm-cost-kit

# Edit config.yaml first, then:
bash setup.sh
source ~/.zshrc
3
manual · one-time · all platforms

Paste the universal output rules into every platform

From core/OUTPUT_RULES.md: paste the User Preferences block into each platform's settings. Takes 2 minutes. Saves tokens on every response, forever.

Paste into: Claude User Preferences · ChatGPT Custom Instructions · Gem Instructions
Respond concisely. Lead with the answer, not the reasoning.
No openers (Great!, Sure!, Certainly!). No closers.
Tables over prose for comparisons.
One recommendation, not a menu of options.
If I ask a yes/no question, answer it first.
💡These 5 rules eliminate 20–60 tokens of pure overhead from every response. At 10 responses/session × 200 sessions/year, that's millions of saved tokens.
4
manual · one-time

Set up context areas on each platform

The same contexts defined in config.yaml map to each platform's native scoping mechanism:

Claude

Sidebar → Projects → New Project. Paste platforms/claude/CLAUDE.md content into Custom Instructions. Create one project per context area. Install platforms/claude/SKILL.md in Cowork → Skills.

ChatGPT / OpenAI

Sidebar → Projects → New Project. Paste platforms/openai/SYSTEM_PROMPT.md into project instructions. Or: Explore GPTs → Create → paste into the Custom GPT system prompt for a reusable agent.

Gemini

Sidebar → Gems → New Gem. Paste platforms/gemini/GEM_INSTRUCTIONS.md into the system instructions. Enable only the Google Workspace extensions this Gem needs. Create one Gem per context.

Snippets

Copy-paste. Works everywhere.

thread summarization — works on all platforms — use at turn 15
Summarize this thread in 150 words: key decisions made,
constraints established, open questions, and the single most
important next step. Preserve any code snippets or structured
data as-is. Omit pleasantries and chain-of-thought.

Paste in current thread → copy output → new chat → paste as first message. Reduces context cost by ~95% while preserving everything that matters.

{
  "task": "one-sentence description of what this agent must do",

  },

python — cost-efficient OpenAI API call
from openai import OpenAI

client = OpenAI()

# Route by task type — don't default to the most expensive model
def get_model(task_type: str) -> str:
    return {
        "quick":     "gpt-4o-mini",  # cheap: lookup, classify, format
        "deep_work": "gpt-4o",      # default: analysis, strategy, code
        "escalate":  "o3",          # only after 2 gpt-4o failures
    }.get(task_type, "gpt-4o")

# Load only the tools this call needs
def call_llm(messages, task_type="deep_work", tools=None):
    budgets = {"quick": 300, "deep_work": 800, "build": 1200}
    return client.chat.completions.create(
        model=get_model(task_type),
        max_tokens=budgets.get(task_type, 800),
        messages=messages,
        tools=tools or [],  # never load tools you won't call
    )
python — cost-efficient Gemini API call
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")  # use env var in production

# Route by task type
def get_model(task_type: str) -> str:
    return {
        "quick":     "gemini-1.5-flash-8b",   # cheapest
        "deep_work": "gemini-2.0-flash",     # default
        "escalate":  "gemini-2.5-pro",       # only on 2 failures
    }.get(task_type, "gemini-2.0-flash")

# Only pass tools if the task actually needs them
def call_llm(prompt, task_type="deep_work", tools=None,
             system=None):
    budgets = {"quick": 300, "deep_work": 800, "build": 1200}
    model = genai.GenerativeModel(
        model_name=get_model(task_type),
        system_instruction=system,
        tools=tools or [],
        generation_config=genai.GenerationConfig(
            max_output_tokens=budgets.get(task_type, 800),
            temperature=0.2,
        )
    )
    return model.generate_content(prompt)
